# [PR 170](https://github.com/vig-os/devkit-smoke-test/pull/170) chore: reset 0.3.7 changelog state on dev for smoke-test re-dispatch

## Summary

Removes the polluted duplicate `## [0.3.7]` sections from `dev`'s CHANGELOG (a stray
`## [0.3.7] - TBD` stacked above a prematurely-dated `## [0.3.7](…) - 2026-06-22`),
restoring a clean empty `## Unreleased` above `## [0.3.6]`.

## Why

Recovery for the wedged 0.3.7 smoke-test gate (see #169). With the stale `[0.3.7]`
content present, the `deploy` idempotency check skips re-seeding `Unreleased` and/or
`prepare-changelog prepare` stacks another heading, so a re-dispatched final pass
cannot produce the `## [0.3.7] - TBD` placeholder that `release-core` validation
requires. This reset lets a fresh final dispatch recreate the entry cleanly.

The stale `release/0.3.7` branch was also deleted so `prepare-release` recreates it.

Refs: #169
