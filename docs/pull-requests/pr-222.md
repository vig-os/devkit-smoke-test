# [PR 222](https://github.com/vig-os/devcontainer-smoke-test/pull/222) chore: migrate dependency automation from dependabot to renovate

Closes #221.

Completes the half-finished Dependabot→Renovate migration in this repo.

## Changes

- **Remove `.github/dependabot.yml`** — it was still managing `github-actions` + `pip` + `npm`, double-managing Actions with Renovate and being the sole (stale) manager for Python. The npm entry was already dead (no `package.json`).
- **Broaden `renovate.json` `enabledManagers`** from `["github-actions"]` to `["github-actions", "pep621", "npm"]`, matching the upstream template `assets/workspace/renovate.json` in `vig-os/devcontainer`. `pep621` is the key addition — it's what makes Renovate cover `pyproject.toml` now that Dependabot is gone.

## Context

The upstream devcontainer template already removed `dependabot.yml` and ships Renovate-only. `renovate.json` is a `PRESERVE_FILES` (never-overwrite) file in `init-workspace.sh`, so this repo — scaffolded from a pre-migration template — never picked up the change automatically; it has to be applied here.

The six stale Dependabot PRs this left behind (#136–#140 pip, #152 actions) have been closed as obsolete.

## Impact

Internal tooling only, no user-visible behavior change → no CHANGELOG entry.

## Verification

- `pre-commit run --all-files` green locally (incl. `check json`, `branch-name`).
- Renovate config validated in CI.
