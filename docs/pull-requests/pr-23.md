# [PR 23](https://github.com/vig-os/devkit-smoke-test/pull/23) fix(ci): grant pull-requests write for dispatch caller

## Description

Repository dispatch triggers were failing at startup because the `dependency-review` job in `ci.yml` requests `pull-requests: write` at the job level, but both the `ci.yml` top-level permissions and the `repository-dispatch.yml` caller permissions only granted `contents: read` — making `pull-requests` implicitly `none`. GitHub validates permissions at parse time before evaluating `if` conditions, so the workflow was rejected even though `dependency-review` would be skipped for non-PR events.

  Not actionable. GitHub Actions does not allow job-level `permissions` to exceed the workflow-level ceiling. When the workflow declares only `contents: read`, every other permission (including `pull-requests`) defaults to `none`. A job-level `permissions: { pull-requests: write }` on `ci-bare` would attempt to escalate beyond that ceiling and fail — reproducing the exact error this PR fixes.

The workflow-level `pull-requests: write` is the minimum required to raise the effective ceiling for the reusable `ci.yml` call.
