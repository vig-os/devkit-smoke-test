`gh workflow run` is authenticated with `${{ github.token }}` here, but the rest of this workflow intentionally uses GitHub App tokens (Commit/Release) for privileged operations. Using `GITHUB_TOKEN` for workflow dispatch can fail in repos/orgs where the token is restricted (repo setting caps it) and diverges from the established pattern of using the Release App token for `gh workflow run`. Consider switching `GH_TOKEN` to `${{ steps.release-app-token.outputs.token }}` (or a similar resolved token) so dispatch is consistent and reliable.
