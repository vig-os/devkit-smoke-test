# [PR 14](https://github.com/vig-os/devkit-smoke-test/pull/14) ci: wire smoke-test dispatch to run RC CI variants

## Description

Wire smoke-test RC orchestration for issue #13 by upgrading `repository_dispatch` handling and making both CI workflows reusable:

- `repository-dispatch.yml`: replace stub with validate/orchestrate/summary flow
- `ci-container.yml`: add `workflow_call` input `image-tag` (default `latest`)
- `ci.yml`: add `workflow_call` trigger

This enables dispatch-driven smoke execution against a specific RC image tag while preserving existing PR/manual behavior.

## Changes Made

1. `.github/workflows/ci-container.yml`
   - Added `workflow_call` trigger with:
     - `inputs.image-tag` (string, default `latest`)
   - Updated both job container images to:
     - `ghcr.io/vig-os/devcontainer:${{ inputs.image-tag || 'latest' }}`

2. `.github/workflows/ci.yml`
   - Added `workflow_call` trigger to allow reusable invocation from orchestration workflows.

3. `.github/workflows/repository-dispatch.yml`
   - Replaced listener stub with:
     - `validate` job (extracts/validates `client_payload.rc_tag`, sets output)
     - `ci-bare` job (uses `./.github/workflows/ci.yml`)
     - `ci-container` job (uses `./.github/workflows/ci-container.yml` with `image-tag`)
     - `summary` job (aggregates and fails on any non-success result)
