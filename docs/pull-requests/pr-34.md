Workflow-level `permissions: contents: write` grants the default `GITHUB_TOKEN` write access in *all* jobs, including `deploy` where a remote `install.sh` is executed. For least-privilege and to reduce blast radius, set workflow permissions to `{}` (or at least `contents: read`) and grant `contents: write` only to the job that actually needs it (likely `publish-release`).
