# [PR 223](https://github.com/vig-os/devkit-smoke-test/pull/223) chore: remove orphaned renovate-changelog workflow

Refs #221. Follow-up to #222 (the removal commit landed one minute after #222 merged, so it needs its own PR).

## Change

Remove `.github/workflows/renovate-changelog.yml`. The devcontainer template split this into `renovate-changelog-build.yml` + `renovate-changelog-commit.yml` (both already present here) and removed the single file. Because the scaffold rsync does not `--delete`, the superseded single file lingered — duplicating the changelog-append run on every Renovate PR.

## Context

Same orphaned-file class as the `dependabot.yml` removed in #222: the template *deleted/renamed* a file, but re-scaffolds only copy template files (never delete), so the old one persists until removed by hand. A drift audit confirmed these were the **only** two orphans — all 91 other managed template files are already in sync.

## Impact

Internal CI tooling only, no user-visible change → no CHANGELOG entry. `pre-commit` green locally.
